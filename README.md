OncoVisionX: Uncertainty-Aware Skin Cancer Triage CDSS

OncoVisionX is an Enterprise Clinical Decision Support System (CDSS) for dermoscopic skin lesion triage. It is architected to solve three critical deployment gaps in medical AI:

Multimodal Reasoning: Fuses visual features (ConvNeXt-Tiny) with clinical metadata (Age, Sex, Anatomical Site).

Formal Safety Guarantees: Utilizes Temperature-Scaled Split Conformal Prediction to issue a "DEFER TO DOCTOR" signal when the mathematical coverage guarantee (90%) is violated.

Out-of-Distribution (OOD) Rejection: Uses a FAISS vector database to detect and reject non-dermoscopic images via visual-only cosine similarity.

🏗️ System Architecture

The system operates on a Dual-Service Microservice Architecture to isolate ML compute workloads from web I/O operations.

[Browser] → Next.js (React 19) Clinical Dashboard
   ↓ (multipart/form-data)
[API Gateway] → Node.js / Express (JWT, MongoDB, Saliency Extraction)
   ↓ (Internal HTTP)
[ML Engine] → Python / FastAPI (PyTorch, ConvNeXt, FAISS, Conformal Predictor)


📂 Repository Structure

This monolithic repository contains four distinct domains:

1. ml_core/ (Research & Model Training)

The PyTorch research environment used to train and calibrate the models.

Highlights: Inverse-weighted Focal Loss ($\gamma=1.5$), Albumentations pipeline, FAISS Index generation, LBFGS Temperature Scaling.

Outputs: Generates the multimodal_convnext_tiny_best.pth weights, visual_ood_index.faiss, and calibration.json.

2. ml_service/ (FastAPI Inference Engine)

A stateless Python microservice that executes the 10-stage SkinTriagePipeline.

Highlights: Pydantic v2 strict schema enforcement, Lifespan context managers (loads 330MB weights into RAM exactly once), 3-pass Test-Time Augmentation (TTA), and Grad-CAM++ generation.

Endpoint: POST /api/v1/predict

3. backend/ (Node.js API Gateway)

An Express.js REST API that handles all clinical routing, authentication, and database persistence.

Highlights: Role-Based Access Control (RBAC), UUIDv4 concurrency safety (eliminates E11000 Mongo errors), 120s Axios timeouts for CPU-mode ML inference, and a Base64-to-Filesystem saliency map extraction pipeline to prevent MongoDB document bloat.

4. frontend/ (Next.js Clinical Dashboard)

A responsive, physician-first React 19 dashboard.

Highlights: TanStack React Query for server state, TailwindCSS / shadcn/ui primitives, a Traffic-Light Three-Way Decision Hero Banner (ACCEPT/DEFER/REJECT), and a side-by-side Grad-CAM++ interpretability split-view.

🚀 Setup & Installation

Prerequisites

Node.js (v20 LTS)

Python (3.10+)

MongoDB (Running locally on port 27017 or a MongoDB Atlas URI)

Step 1: Start the ML Service

The ML microservice requires the pre-trained weights and FAISS index generated by ml_core.

cd ml_service
python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate
pip install -r requirements.txt

# Start the FastAPI engine (Runs on port 8000)
uvicorn app.main:app --reload


Step 2: Start the API Gateway

cd backend
npm install

# Setup Environment Variables
cp .env.example .env
# Ensure ML_API_URL=http://localhost:8000 and configure your MongoDB URI

# Start the Node.js server (Runs on port 5000)
npm run dev


Step 3: Start the Frontend Dashboard

cd frontend
npm install

# Setup Environment Variables
cp .env.local.example .env.local
# Ensure NEXT_PUBLIC_API_URL=http://localhost:5000/api

# Start the Next.js application (Runs on port 3000)
npm run dev

Conformal Coverage: ≥ 90% (Formal Statistical Guarantee)
